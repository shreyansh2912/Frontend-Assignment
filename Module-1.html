Q1. Are the HTML tags and elements the same thing?
Ans:
HTML Tags:

These are the starting and ending parts of an HTML element, consisting of a starting tag and an ending tag . They tell the browser how to display the content and usually exist in pairs

HTML Elements:

These are the content enclosed inside the HTML tags, consisting of a start tag, content, and an end tag. They represent the actual content that users see on the web page


Q2. What are tags and attributes in HTML?
Ans:

In HTML, tags and attributes are two important concepts. HTML tags are used to define the structure of an HTML document and contain instructions on how to display an on-page element. They begin with a < and end with a > and sandwich the content of the element. HTML elements are the on-page content enclosed within the tags. HTML attributes provide additional information about HTML elements and appear within the HTML tags. They are used to describe the characteristics of an HTML element in detail. Attributes are found only in the starting tag and are used to define the character of an HTML element. They generally provide additional styling to the element. While the elements tell the browser what to display, the attributes define how they will behave

Q3. What are void elements in HTML?
Ans:

Void elements in HTML are a special category of elements that are defined as having no content and no closing tag. They are self-contained and do not require any additional information to be rendered correctly. Void elements can have attributes but cannot have any child elements. Some examples of void elements include br, hr, img, input, link, and meta. In contrast to void elements, regular elements have both start and end tags and can contain content. Void elements are parsed as standalone elements in HTML, and their closing tags are not required

Q4. What are HTML Entities?
Ans:

HTML entities are special characters that are used to display reserved characters in HTML. Reserved characters are those that have a special meaning in HTML, such as the less than sign (<) and greater than sign (>). If you use these characters in your HTML text, the browser might mix them with tags. To avoid this, you can use entity names or entity numbers to display reserved HTML characters. Entity names look like this: &entity_name; and entity numbers look like this: &#entity_number;. For example, to display a less than sign (<), you can write < or < 1. HTML entities can also be used to display characters that are not present on a normal keyboard, such as mathematical symbols, accented letters, and emojis

Q5. What are different types of lists in HTML?
Ans:

In HTML, there are three main types of lists: unordered lists, ordered lists, and description lists. Each type serves a specific purpose and can be used to present information in a logical or linear form
1. Unordered List (UL): An unordered list is used to create a list of related items, in bulleted or unordered format. It starts with the <"ul"> tag, and each list item starts with the <"li"> tag. By default, list items are marked with bullets (small black circles)
2. Ordered List (OL): An ordered list is used to create a list of items with a specific order or sequence. It starts with the <"ol"> tag, and each list item starts with the <"li"> tag. By default, list items are marked with numbers
3. Description List (DL): A description list is a type of list where each item has a description. It is also known as a definition list. The <"dl"> tag is used to create a description list, the <"dt"> tag defines the item (term), and the <"dd"> tag describes each item in the list

Q6. What is the ‘class’ attribute in HTML?
Ans:

In HTML, the class attribute is used to define a class for an HTML element. It is used to group elements with similar characteristics and apply styles to them. The class attribute is used in conjunction with CSS to style HTML elements. Multiple classes can be assigned to a single element by separating them with a space. The class attribute is a global attribute, which means it can be used with any HTML element. The class attribute is defined in the opening tag of an HTML element and is followed by the class name. For example, <"p class="example"> assigns the class name "example" to the paragraph element

Q11 . What is the difference between a block-level element and an inline element?
Ans:

The difference between a block-level element and an inline element in HTML is as follows:
Block-level elements: They always start on a new line and take up the full width available. They can have top and bottom margins, and their width can be modified using CSS. Examples of block-level elements include <"div>, <"p>, <"h1> to <"h6>, <"ol>, <"ul>, and <"li>.
Inline elements: They do not start on a new line and only take up as much width as necessary. They do not have top and bottom margins, and their width is determined by the content. Examples of inline elements include <"span>, <"a>, <"strong>, <"em>, <"img>, and <"input>.
The distinction between block-level and inline elements is important for understanding how elements are displayed and how they interact with other elements on a web page

Q13 . What is the use of an iframe tag?

Q18 . What is SVG?
Ans:

SVG, or Scalable Vector Graphics, is an XML-based vector image format for two-dimensional graphics with support for interactivity and animation. It is a widely used format for creating high-quality, resolution-independent images for the web. SVG images can be created and edited with any text editor, and their properties can be manipulated using CSS and JavaScript. Unlike raster image formats, such as JPEG or PNG, SVG images are defined mathematically, allowing them to be scaled to any size without losing quality. This makes SVG a preferred format for icons, logos, illustrations, and other graphics on the web.


Q19 . What is difference between HTML and XHTML?
Ans:

HTML (Hypertext Markup Language) and XHTML (Extensible Hypertext Markup Language) are both markup languages used for creating and displaying web pages. The main difference between them is the syntax and structure. HTML is more lenient in its syntax, while XHTML has a more strict syntax and follows XML rules. XHTML is considered the successor to HTML, and its strict syntax makes it more suitable for future technologies and advancements. XHTML is an XML-based version of HTML that follows stricter rules and syntax, making it easier for web pages to be parsed and processed by both humans and machines. XHTML is also designed to be XML-compliant, making it more accessible, readable, and compatible with all web browsers. In summary, HTML is more forgiving and allows certain coding practices to be less strict, while XHTML enforces a stricter set of rules and requires well-formed XML syntax, aiming to combine the flexibility of HTML with the rigorousness of XML
