Q3. What is the difference between CSS2 and CSS3?

Ans. CSS2 and CSS3 are both versions of the Cascading Style Sheets (CSS) language, but there are some key differences between the two.
CSS2 was released in 1998 and is considered to be a stable and mature version of the language. It includes features such as support for media types, cascading and inheritance, and selectors.
CSS3, on the other hand, is the latest version of the language and was first introduced in 1999. It includes new features such as animations, transitions, and new selectors. It also includes support for responsive design and new layout modules such as Flexbox and Grid. CSS3 also allows for better accessibility and more control over typography.
Additionally, CSS3 is not a singular monolithic release, it is a series of individual modules that are updated and released as they reach a certain level of maturity.
In summary, CSS3 includes new features and capabilities that were not present in CSS2, making it more powerful and versatile for web design and development.

Q4. Name a few CSS style components

Ans. CSS style components are reusable pieces of code that can be applied to HTML elements to give them a consistent look and feel. Some examples of CSS style components are:
1. Selectors: These are the patterns that match one or more elements in the HTML document. For example, h1 selects all <'h1> elements, and .container selects all elements with the class attribute container.
2. Properties: These are the aspects of the element that can be styled, such as color, font-size, margin, border, etc. Each property has a specific syntax and a set of possible values.
3. Values: These are the actual settings for the properties, such as red, 16px, auto, solid, etc. Values can be keywords, numbers, units, colors, functions, or other expressions.
4. Declarations: These are the pairs of properties and values that are enclosed in curly braces and separated by semicolons. For example, color: red; is a declaration that sets the color property to red.
5. Rules: These are the combinations of selectors and declarations that specify how the selected elements should be styled. For example, h1 { color: red; } is a rule that applies the color red to all <'h1> elements.

Q5. What do you understand by CSS opacity?

CSS opacity is a property that sets the transparency of an element. It describes the degree to which the content behind an element is hidden, and is the opposite of transparency. A value of 1 is fully opaque (not transparent at all), while a value of 0 is fully transparent. The opacity property can be set using values from 0.0 (fully transparent) to 1.0 (fully opaque), or using a percentage from 0% to 100% This property can be applied to various elements, such as images and text, to control their transparency

Q8. What is the use of the background-position property?

Ans. The background-position property in CSS is used to set the initial position of a background image within its container. It allows you to control the placement of the background image. The property can take various values, including keywords (such as top, bottom, left, right, and center), length values, and percentage values. For example, background-position: top right; will position the background image at the top right corner of the element. You can also use percentage values like background-position: 50% 25%; to position the background image at a specific percentage of the container's width and height. This property provides flexibility in positioning background images to achieve the desired visual layout

Q9. Which property controls the image scroll in the background?

Ans. The property that controls the image scroll in the background is the background-attachment property in CSS. It sets whether a background image scrolls with the rest of the page or is fixed. It has three values: scroll (the background image will scroll with the page), fixed (the background image will not scroll with the page), and local (the background image will scroll with the element's contents)

Q10. Why should background and color be used as separate properties?

Ans. The background and color properties should be used separately for several reasons. First, it makes style sheets more legible for both humans and machines. The background property is already complex, and combining it with the color property would make it more so. Second, the color property inherits, but background doesn't, which could lead to confusion. Therefore, to avoid conflicts with user style sheets and for better readability, it is recommended to specify the background and color properties separately

Q14. What is embedded style sheets?

Q15.What are the external style sheets?

Ans. External style sheets are separate files with a .css extension that are referenced in the main HTML document. They are used to define styles for multiple HTML pages, making it easier to maintain and update the design of a website. To use an external style sheet, follow these steps:
1. Create a .css file with the desired styles. The file should not contain any HTML tags.
2. Save the .css file in the same folder as your HTML files or in a dedicated folder for your CSS files.
3. In the <'head> section of your HTML document, add a <'link> tag to reference the external style sheet. The href attribute should point to the location of the .css file, and the rel attribute should be set to "stylesheet".

Q16.What are the advantages and disadvantages of using external style sheets?

Ans. The advantages of using external style sheets include:
1. Consistency: Styles can be applied consistently across multiple pages.
2. Maintainability: Changes to the design can be made by modifying a single file, making it easier to maintain.
3. Caching: External style sheets can be cached by the browser, allowing them to be reused across multiple pages, which can improve loading times. The disadvantages of using external style sheets are:
1. Additional Download: An extra download is needed to import style information for each document, which can impact loading times.
2. Rendering Dependency: Until the external style sheet is loaded, it may not be possible to render the document.
3. Viability for Small Projects: For a small number of style definitions, using an external style sheet may not be viable

Q17. What is the meaning of the CSS selector?

Ans. A CSS selector is a pattern of elements and other terms that tell the browser which HTML elements should be styled. It is the first part of a CSS rule and is used to select the HTML element(s) you want to style. CSS selectors can be divided into five categories: simple selectors (select elements based on name, id, class), combinator selectors (select elements based on a specific relationship between them), pseudo-class selectors (select elements based on a certain state), pseudo-elements selectors (select and style a part of an element), and attribute selectors (select elements based on an attribute or attribute value). CSS selectors can be used to target specific elements on a web page and apply styles to them

These media types allow for the specification of different styles for different output devices and media

Q19. What is the rule set?

Ans. A CSS rule set, also known as a rule, consists of a selector and a declaration block. The selector points to the HTML element you want to style, and the declaration block contains one or more declarations separated by semicolons. Each declaration includes a CSS property name and a value, separated by a colon. Multiple CSS declarations are separated with semicolons, and declaration blocks are surrounded by curly braces. For example, in the rule set p { color: red; }, the selector is p, and the declaration block contains the color property with a value of red
